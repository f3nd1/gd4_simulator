# Spreadsheet Evidence Skill

## Spreadsheets are common and important evidence

Excel (.xlsx), CSV, and Google Sheets files are among the most common evidence types submitted by institutions. They should never be flattened into anonymous text. Always preserve the file name, sheet name, column headers, and row ranges when describing spreadsheet content.

## Typical spreadsheet evidence types

- Attendance registers (training, induction, orientation, meetings)
- Financial aid / FPS tracker with student names, amounts, eligibility flags
- Complaint logs with date, complainant, category, status, resolution
- Student progress tracking sheets
- Survey response exports with question columns and response rows
- Key Performance Indicator (KPI) dashboards
- Action item trackers with owner, due date, status, closure date
- Audit finding logs

## How to assess a spreadsheet

**Preserve structure**: When extracting text from a spreadsheet, keep: File name, Sheet name, Column headers (first row), Row data with row numbers, and note if rows were capped. Do NOT flatten into a paragraph — the column structure is the evidence.

**Proving implementation**: A spreadsheet proves implementation only if it contains ROWS of actual completed activities. A template with headers but no data rows does NOT prove implementation. A spreadsheet with only a few rows for a multi-month period is Weak evidence. Assess: How many rows? What date range? Which cohort or group?

**Proving outcomes**: A spreadsheet proves outcomes only if it contains aggregated results, measured trends, target vs actual comparisons, or before/after data. A log of activities (e.g. "training held on date X") is implementation evidence, not outcome evidence. Outcome evidence requires: measured result, a target or baseline to compare against, or a trend over time.

**Proving review**: A spreadsheet proves review only if it contains decision columns (approved / rejected / deferred), follow-up owner, closure date, or a status column that shows items were actioned. A list of action items with all statuses blank is NOT review evidence.

## Empty or header-only spreadsheets

An empty spreadsheet (headers only, no data rows) provides zero evidence of any dimension. Do not treat it as evidence of planning or approach. Flag it as: "Spreadsheet submitted but contains no data rows — headers only."

## Activity counts vs outcome evidence

The number of activities logged (e.g. "45 students attended") is implementation evidence. It is NOT outcome evidence unless accompanied by a quality measure (pass rate, satisfaction score, improvement metric, comparison to target).

## AI assessment of spreadsheets

When assessing a spreadsheet chunk, evaluate:
- Row coverage: How many records? Over what period?
- Cohort completeness: Is this all students / all staff, or a sample?
- Data completeness: Are key columns filled in or mostly blank?
- Recency: Is this from the current audit period?
