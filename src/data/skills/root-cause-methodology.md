# Root Cause Methodology — How to Identify the Real Cause

A finding's root cause must explain **WHY** the gap exists at a systemic level — not describe the gap itself. Superficial root causes lead to superficial corrective actions that recur. Apply the discipline below.

## The symptom / immediate cause / root cause hierarchy

| Level | What it looks like | Example |
|---|---|---|
| **Symptom** (what you observed) | The missing record, the unfiled document, the unmet metric | "Pre-course counselling forms missing for 8 students" |
| **Immediate cause** (the proximate trigger) | The process step that failed | "Admissions staff did not complete the form before enrolment" |
| **Root cause** (the systemic reason) | Why no system caught or prevented the immediate cause | "The admissions SOP does not include a mandatory pre-counselling checkpoint; no supervisor review of files before enrolment confirmation" |

**The AI must reach Level 3 (root cause).** Writing "staff did not follow procedure" is Level 2 at best. Ask WHY three times.

## 5-Why technique (apply for every finding)

1. Why did the gap occur? → [immediate cause]
2. Why did that happen? → [underlying cause]
3. Why did that go undetected / unpreveneted? → [systemic cause = root cause]

Example chain for a missing-moderation gap:
1. Why were scripts not moderated? → "No second marker was assigned."
2. Why was no second marker assigned? → "The assessment plan does not designate a moderator role."
3. Why is there no moderator role in the plan? → "The assessment policy was written without a moderation section; the Academic Board has never reviewed or approved a moderation procedure." ← **This is the root cause.**

## Common root-cause categories in GD4 audits

- **Policy/SOP gap**: The procedure was never written, is too generic, or does not cover this scenario. Corrective action = write/revise the policy; preventive action = approve at Academic/Management Board with a scheduled review cycle.
- **Training gap**: Staff know the requirement exists but not how to implement it correctly. Corrective = targeted training with sign-off; preventive = onboarding and annual refresher protocol.
- **Governance gap**: No one is accountable for monitoring compliance with this requirement. Corrective = assign a named owner; preventive = add to a standing agenda at Management Review or Academic Board.
- **Data-collection gap**: The institution has a process but collects no outcome or quality data to verify it works. Corrective = define metrics and start collecting; preventive = build into the management review dashboard.
- **Review gap**: The process existed but was never evaluated for effectiveness. Corrective = conduct an evaluation now; preventive = scheduled periodic review with documented output.

## Root cause language for findings

- AVOID: "Management did not monitor this area sufficiently."
- USE: "No formal monitoring mechanism (e.g. a pre-submission file-completeness check or a supervisor sign-off) was built into the [process name]. As a result, the gap was not detectable until the internal audit."

- AVOID: "Staff were not aware of the requirement."
- USE: "The induction programme (last reviewed [date]) does not include training on [requirement]. No subsequent briefing record confirms that affected staff received the updated guidance."

## Corrective vs preventive action (do not conflate)

- **Corrective** = fixes the specific existing gap now. Time-bound. Names the document/record to be created and the person responsible.
- **Preventive** = changes the system so the gap cannot recur. Usually a policy change, a standing review item, or a checkpoint in a workflow. Must be different from the corrective action — "complete the missing forms" is not a preventive measure.
