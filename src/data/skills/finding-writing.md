# Finding Writing — Root Causes and Corrective Actions

Good findings name the SYSTEM failure, not just the symptom. A finding that says "documentation is missing" tells the institution nothing about why it's missing or how to prevent it recurring. A finding that says "no formal ownership has been assigned for maintaining and reviewing this policy, so it is created ad-hoc when needed and lapses between audit cycles" is actionable.

## The three-level root cause

Ask "why?" three times:

1. **What happened?** (the observed gap) — "No pre-course counselling record was found for 8 of 23 student intakes reviewed."
2. **Why did it happen?** (the immediate cause) — "The admissions counsellor completes the record manually after each session, but there is no prompt or system check before processing the student contract."
3. **Why was the system unable to prevent it?** (the system root cause) — "The student contract workflow does not include a mandatory pre-course counselling completion step, so the process is bypassed without detection."

Write the root cause at level 3. Level 1 is the observation (already stated in the finding). Level 2 is a symptom, not a cause.

## Common weak root causes — and how to improve them

| Weak (avoid) | Better |
|---|---|
| Lack of documentation | No formal ownership assigned for maintaining Policy X; it is revised on request rather than on a scheduled cycle |
| Staff not trained | Training needs analysis does not include new joiners in their first 90 days; the induction programme covers procedures but not this requirement |
| Process not followed | The workflow system does not enforce completion of Step Y before Step Z is accessible; staff are able to bypass it without a recorded override |
| No review conducted | Management Review agenda template does not include a standing item for reviewing this process; it is only reviewed if a complaint prompts it |
| Insufficient evidence collected | Data collection form does not capture the outcome metrics required (only activity counts); the form predates the current GD4 requirement |

## Corrective action — fix the specific gap now

- Time-bound and verifiable: "Complete pre-course counselling for all current enrolled students and record using Form ADM-05 by [date]"
- Addresses the specific instance: "Retrieve and file the missing records for the 8 students identified above, or document that the counselling cannot be retrospectively evidenced"
- Must be completable and evidence-able: the auditor must be able to check at re-audit that it was done

## Preventive action — change the system

- Prevents the same gap recurring for future cohorts/cycles
- Changes a PROCESS or SYSTEM, not just trains people (training-only preventive actions are weak)
- Examples:
  - "Modify the student intake workflow to add a mandatory pre-course counselling completion checkbox before the contract signing step is unlocked"
  - "Add a standing agenda item to the Management Review template to verify pre-course counselling completion rate each semester"
  - "Assign the role of Policy Owner for all student protection policies to the Admissions Manager, with a scheduled annual review by June each year"

## The dimension-specific structure

Tailor the root cause to the APSR dimension that fell short:

- **Approach gap:** Root cause = the policy/procedure has not been formally established, approved, or is too generic. Corrective = draft and approve the specific procedure. Preventive = establish a policy calendar with owners and review dates.
- **Processes gap:** Root cause = the policy is documented but there is no systematic mechanism ensuring it is followed. Corrective = gather or reconstruct the missing records. Preventive = build the process step into a workflow, form, or system check.
- **Outcomes gap:** Root cause = the process runs but outcomes are not measured. Corrective = collect and analyse the outstanding outcome data. Preventive = add outcome KPIs to the data management system with reporting cadence.
- **Review gap:** Root cause = there is no scheduled review for this process. Corrective = schedule and conduct a review now. Preventive = add to Management Review agenda as a standing item with a named reviewer.
